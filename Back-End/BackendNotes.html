Real Life: Customer → Security Guard → Dukaan Request → Middleware → Controller Security guard check karta hai — phir andar jane deta hai! Simple Example: // Middleware = function (req, res, next) const myMiddleware = (req, res, next) => { console.log("Request aaya!") next() // Aage bhejo — zaroori! } app.use(myMiddleware) // Sab routes pe lagao next() = "Theek hai — aage jao!" next() nahi likha = Request ruk jayegi! 3 Types: 1 — Application Middleware — Sab routes pe: app.use(express.json()) // Yeh bhi middleware hai! app.use(cors()) // Yeh bhi! // Apna middleware app.use((req, res, next) => { console.log(`${req.method} ${req.url}`) next() }) 2 — Route Middleware — Sirf ek route pe: // 2. Sirf ek specific route mein shamil karna app.get('/watch-movie', checkPremiumUser, (req, res) => { // Yeh controller sirf tab chalega jab checkPremiumUser 'next()' chalaye ga res.send("Aap premium movie dekh rahe hain! 🍿"); }); // Is normal route par koi checking nahi hogi kyunki yahan middleware nahi lagaya app.get('/free-videos', (req, res) => { res.send("Yeh free video sab ke liye hai."); }); 3 — Error Middleware — Errors pakadna: // 4 parameters hote hain — err pehle! app.use((err, req, res, next) => { res.status(500).json({ message: err.message }) }) Flow: Request ↓ logger.js → Log karo ↓ auth.js → Token check karo ↓ Controller → Kaam karo ↓ Response Topic 6 — MongoDB + Mongoose 🎯 MongoDB Kya Hai: Database = Data store karne ki jagah!
