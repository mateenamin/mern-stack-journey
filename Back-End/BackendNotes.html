(Asan Lafzon Mein) Authentication ka seedha sa matlab hai: "User ki pehchan karna (Identity Verify karna.)" Yeh back-end ka woh system hota hai jo yeh check karta hai ke jo banda website par login karne ki koshish kar raha hai, kya woh sach mein wahi hai ya koi hacker hai. src/ └── controllers/ └── auth.controller.js <-- 🔑 User ko Register aur Login karwane ka kaam (Authentication) is file mein hota hai. Authentication vs JWT Authentication ek kaam hai: Jab user website par apna Email aur Password daalta hai toh controller database mein ja kar check karta hai ke password sahi hai ya galat. Is poore check karne ke process ko hum Authentication kehte hain. JWT ek tool ha: Jab authentication kamyab ho jati hai (yani back-end maan leta hai ke user sahi hai), toh controller user ko baar-baar password daalney ki takleef se bachane ke liye ek digital pass de deta hai, jise hum JWT Token kehte hain. JWT Kya Hai? (The Digital Pass) JWT ka matlab hai JSON Web Token Aasan lafzon mein, yeh bilkul ek (Hand band) ki tarah hai. (1)Jab aap ticket counter par paise dete hain, toh woh aapko ek band baandh dete hai (2) Ab aap park ke andar jitne bhi jhoole (pages) par jayenge, wahan khada guard (middleware) aap se baar-baar aapka identity card ya paise nahi mangega. Woh bas aapka band (token) dekhega aur kahega: "Aage jao." Back-end mein bhi bilkul yahi hota hai. Jab user ek baar sahi email aur password daal kar login ho jata hai, toh controller use ek JWT Token (ek lamba encrypted text) bhej deta hai. Yeh Code Dikhta Kaisa Hai? Topic 10 bcrypt Kya Hai Aur Iski Zaroorat Kyun Hai? Jab bhi koi user aapki website par naya account banata hai (Sign-up karta hai), toh woh apna ek password set karta hai—jaise Mateen786 Agar aap is password ko database mein bilkul isi tarah saaf-saaf (Plain Text) save kar denge, toh iske do bade nuksaan hain: Database Leak: Agar kisi hacker ne aapka MongoDB Atlas ka database hack kar liya, toh use sab ke real passwords mil jayenge. Internal Access: Aap (as a developer) ya aapka koi team member bhi database khol kar kisi ka bhi password dekh sakta hai, jo ke bilkul galti hai. Is masle ko hal karne ke liye hum bcrypt (npm package) use karte hain. Yeh ek One-Way Hashing Algorithm hai. Iska kaam hai real password ko ek aise ajeeb aur lambe khufia code mein badal dena jise koi insaan ya computer decode (read) na kar sake—jaise $2b$10$X7vRt... Folder Structure: bcrypt ko use karne ke liye hamein koi naya folder nahi banana padta. Yeh hamare pehle se maujood folders ke andar hi kaam karta hai: src/ ├── models/ │ └── user.model.js <-- ⚡ Tarika A: Mongoose Schema ke andar automatic lagana └── controllers/ └── auth.controller.js <-- ⚡ Tarika B: Controller ke andar direct lagana Industry mein ise use karne ke 2 tarike hain. Hum dono ka sirf concept aur rasta dekhte hain Tarika A: Model Ke Andar (Professional Method) Professional coders ise direct user.model.js ke andar set kar dete hain jise Pre-Save Hook kehte hain. Iska matlab hota hai ke jab bhi controller user ka data save karne lagega, model automatic database mein jaane se pehle password ko khud hi hash (chupa) dega. Controller ko fikar karne ki zaroorat hi nahi hoti.
